系统管理（sys）

核心模块权限中心系统配置

功能入口

  • 用户：/sys/user/*（list/info/password/save/update/delete）
  • 配置：/sys/config/*（list/basic/info/save/update/delete/data）
  • 菜单与角色：/sys/menu/*、/sys/role/*、/sys/dept/*
  • 运维工具：/sys/cache/*、/sys/redis/*、/sys/thread/*、/sys/queue/*

核心功能点

  • 权限模型：角色-菜单-用户绑定
  • 组织模型：部门层级与数据范围联动
  • 系统配置：参数化管理与运行时读取
  • 日志审计：关键行为可追踪

文档补充清单

  • 补“新增管理员账号并授权”的完整流程
  • 补“新增菜单 + 绑定权限 + 挂角色”的流程图
  • 补常见问题：菜单不显示、按钮无权限、数据越权检查

关键配置项（来自 sys_config.param_key）

配置键 默认值 说明
LOGIN_AUTHENTICATION oauth2:{clientId} 登录授权模式：localhost / oauth2:clientId / shell:clientId
TOKEN_GENERATE_STRATEGY current Token 策略：current(复用有效token) / new(每次新建)
LOGGER_LEVEL INFO 动态日志级别：ALL/TRACE/DEBUG/INFO/WARN/ERROR/OFF
DEBUG_MODE false 调试模式开关

API 示例（真实路径）

GET /sys/user/list?page=1&limit=10
POST /sys/user/password?password=oldPass&newPassword=newPass
POST /sys/config/update {"paramKey":"LOGGER_LEVEL","paramValue":"DEBUG"}
GET /sys/menu/nav

返回结构示例

{"code":0,"msg":"success","page":{"list":[...],"currPage":1,"totalPage":3,"totalCount":24}}

权限码清单

sys:user:list|info|save|update|delete
sys:config:list|info|save|update|delete
sys:menu:list|select|info|save|delete
sys:role:list|select|info|save|delete
sys:dept:list|select|info|save|update|delete
sys:dict:list|info|save|update|delete, sys:category:list|info|save|update|delete, sys:log:list

最小可跑通用例

  1. 以管理员登录系统，确认角色包含 sys:user:list 与 sys:config:list。
  2. 访问 /modules/sys/user，确认用户列表正常加载。
  3. 调用 GET /sys/user/list?page=1&limit=10，确认返回 code=0 且包含 page.list。
  4. 调用 POST /sys/config/update 更新一项配置（如 LOGGER_LEVEL），确认返回成功。

常见故障排查模板

现象 可能原因 处理建议
菜单看不到 角色未绑定菜单或缓存未刷新 检查角色权限并清理权限缓存
配置更新未生效 修改了错误的 paramKey 或类型不匹配 检查 /sys/config/update 请求体与配置类型
接口返回“权限不足” 缺少 @RequiresPermissions 对应权限 检查角色是否包含如 sys:user:list 等权限码
