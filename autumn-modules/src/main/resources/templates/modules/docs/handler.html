Handler 机制（autumn-handler）

本章用于说明 Autumn 在分模块开发模式下，如何通过 Handler 接口把“模块业务能力”接入“框架主流程”。

为什么需要 Handler

  • 将框架流程与模块实现解耦：框架定义接口，模块按需实现。
  • 支持可插拔扩展：默认接口通常通过 @ConditionalOnMissingBean 提供兜底。
  • 支持顺序控制：部分 Handler 通过 @Order 参与优先级编排。
典型场景：模块希望定制登录页、鉴权链、资源解析、插件装载、集群用户映射等能力，不改核心框架代码即可接入。

按职责分组的核心 Handler

分组接口示例用途
页面与视图 PageHandler / ViewHandler 定制 login/index/main/error 等页面路由与视图解析。
安全与拦截链 FilterChainHandler / InterceptorHandler / WallHandler 定义过滤链、拦截器匹配规则、墙策略关联点。
配置与初始化 InitHandler / RefreshHandler / ResolverHandler 初始化、刷新、参数解析等生命周期扩展点。
插件与资源 PluginHandler / ResourceHandler / LoadHandler 插件装载、资源暴露和加载策略扩展。
集群与用户映射 ServiceHandler / UserHandler / PermissionHandler 服务地址、跨系统用户映射、权限映射能力。
检索与监听 SearchHandler / LoginListenerHandler 统一检索入口与登录事件监听扩展。

分模块开发中的调用关系（示例）

  1. 模块实现某个 Handler（例如 PageHandler）并注册为 Spring Bean。
  2. 框架在执行流程节点时按接口查找实现（无实现时使用默认行为）。
  3. 若有多个实现，按 @Order 或框架策略挑选有效实现。
  4. 最终输出进入 Controller/Filter/View 等主流程执行。
模块实现 Handler -> Spring 注入 -> 框架节点回调 -> 返回控制结果（路由/配置/映射）

项目实现地图（当前代码）

Handler接口主要实现类位置作用

推荐阅读顺序（从流程到实现）

  1. 先看 PageHandler 与 DefaultPage，理解页面入口如何被接管。
  2. 再看 InterceptorHandler 系列实现，理解请求链路中的横切处理。
  3. 再看 DomainHandler/CategoryHandler/ClearHandler，理解模块如何介入配置与运维流程。
  4. 最后看 SearchHandler/ResolverHandler，理解通用能力如何被统一抽象。

命名注意点

  • 项目中存在两个 VersionHandler：cn.org.autumn.config.VersionHandler 与 cn.org.autumn.cluster.VersionHandler，阅读和实现时需注意导包。
  • 部分能力通过扩展工厂接口（如 InitFactory.Init、LoadFactory.Load）介入，不完全以 “*Handler” 命名出现。

新增模块接入 Handler 标准模板

以下模板可作为团队统一规范，新增模块时优先采用“最小覆写 + 清晰优先级 + 可验证日志”策略。

模板一：页面路由接入（PageHandler）

package cn.org.autumn.modules.demo.site; import cn.org.autumn.config.PageHandler; import org.springframework.core.annotation.Order; import org.springframework.stereotype.Component; import org.springframework.ui.Model; import jakarta.servlet.http.HttpServletRequest; import jakarta.servlet.http.HttpServletResponse; @Component @Order(100) public class DemoPageHandler implements PageHandler { @Override public String login(HttpServletRequest req, HttpServletResponse resp, Model model) { // 仅在需要接管登录页时返回具体视图；否则返回空字符串交给后续处理 return ""; } }

模板二：请求拦截接入（InterceptorHandler）

package cn.org.autumn.modules.demo.interceptor; import cn.org.autumn.config.InterceptorHandler; import org.springframework.stereotype.Component; import org.springframework.web.servlet.HandlerInterceptor; import java.util.Arrays; import java.util.List; @Component public class DemoInterceptorHandler implements InterceptorHandler { @Override public HandlerInterceptor getHandlerInterceptor() { return new DemoInterceptor(); } @Override public List getPatterns() { return Arrays.asList("/modules/demo/**", "/api/demo/**"); } @Override public List getExcludePatterns() { return Arrays.asList("/api/demo/health"); } }

模板三：参数解析接入（ResolverHandler）

package cn.org.autumn.modules.demo.resolver; import cn.org.autumn.config.ResolverHandler; import org.springframework.core.MethodParameter; import org.springframework.stereotype.Component; import org.springframework.web.bind.support.WebDataBinderFactory; import org.springframework.web.context.request.NativeWebRequest; import org.springframework.web.method.support.HandlerMethodArgumentResolver; import org.springframework.web.method.support.ModelAndViewContainer; @Component public class DemoArgumentResolver implements HandlerMethodArgumentResolver, ResolverHandler { @Override public boolean supportsParameter(MethodParameter parameter) { return parameter.getParameterType().equals(DemoContext.class); } @Override public Object resolveArgument(MethodParameter parameter, ModelAndViewContainer mavContainer, NativeWebRequest webRequest, WebDataBinderFactory binderFactory) { return new DemoContext(); } }

接入验证清单（建议在 PR 中勾选）

  • 实现类已加 @Component，并确认启动扫描路径可覆盖。
  • 若存在多个实现，已明确 @Order 并记录排序理由。
  • 关键方法有最小日志（至少能确认是否命中实现）。
  • 已在文档页登记“接口名 -> 实现类 -> 作用 -> 验证方式”。
  • 已完成一次端到端验证（页面访问/接口调用/返回结果）。

提交前检查（避免常见回归）

检查项风险建议
返回空字符串语义错误可能导致错误抢占或流程短路只有“明确接管”才返回非空，其他情况返回空字符串
拦截路径过宽误伤静态资源与登录流程先限定到模块路径，再逐步扩大
多个实现优先级冲突行为不稳定、环境差异大统一在团队内约定 Order 区间

最小接入步骤

  1. 在业务模块中新建实现类（例如实现 PageHandler）。
  2. 只覆写必要方法（例如 login()、index()）。
  3. 添加 @Component 并设置合适 @Order（若需优先级）。
  4. 启动后访问对应页面，验证是否命中你的实现。
public class CustomPageHandler implements PageHandler { public String login(...) { return "oauth2/login"; } }
