队列体系（queue）

BaseQueueServiceQueueService死信与历史自动启停消费者

功能入口

  • 业务接入基类：cn.org.autumn.service.BaseQueueService
  • 队列内核服务：cn.org.autumn.service.QueueService
  • 运维管理接口：/sys/queue/*（QueueController）
  • 管理页面入口：/sys/queue/ -> templates/queue.html

核心能力说明

  • 支持多种队列类型：MEMORY、REDIS_LIST、REDIS_STREAM、DELAY、PRIORITY。
  • 支持多发送模式：普通、延迟、定时、优先级、批量发送。
  • 支持自动消费者：可自动启动，空闲超时后自动停止。
  • 支持失败处理：重试次数、死信队列、历史消息保留与回查。

关键配置项（QueueConfig）

字段 默认值 说明
queueType MEMORY 队列实现类型
capacity 100000 队列容量上限（内存队列有效）
retries 3 失败重试次数
deadLetter false 是否启用死信队列
history 100 历史消息保留数量（已消费）
auto / idleTime / idleUnit true / 30 / SECONDS 自动启停及空闲超时策略

权限码清单

当前控制器未声明 @RequiresPermissions，且标注了 @SkipInterceptor。 建议生产环境通过网关/IP 白名单/管理员鉴权二次加固后再开放。

最小可跑通用例

  1. 访问 /sys/queue/，确认可以看到队列管理页。
  2. 调用 POST /sys/queue/create 创建测试队列（建议开启 deadLetter）。
  3. 调用 POST /sys/queue/send 发送一条 JSON 消息。
  4. 调用 GET /sys/queue/messages/{name} 或页面“消息列表”确认消息入队。
  5. 调用 POST /sys/queue/consumer/start/{name} 启动消费者，再观察 stats 和 history 变化。

常见故障排查

现象 可能原因 处理建议
消息堆积不消费 消费者未启动或被空闲自动停止 检查 /sys/queue/consumers，必要时手动 start consumer
消息重复失败 业务处理异常持续发生 查看历史消息错误，修复后执行 /deadletter/retry/{name}
队列配置更新不生效 参数类型不正确或队列不存在 先用 /config/{name} 校验，再执行 /config/update/{name}
