超级位置模型（spm）

用于埋点建模与访问日志分析，支持业务点击/访问统计场景。

功能入口

  • 模型管理：/spm/superpositionmodel/list|save|update|delete
  • 访问日志：/spm/visitlog/list|info
  • 页面资源映射：/admin、/、/{value}（SpmController）

建议补充

  • 埋点字段规范（事件名、来源、位置、上下文）
  • 指标口径定义（PV/UV/转化）
  • 从埋点到报表的全链路示例
建议补充“埋点版本变更策略”，确保老数据可兼容解析。

关键配置项（来自 sys_config.param_key）

配置键 默认值 说明
MENU_WITH_SPM 1 菜单是否启用 SPM 模式统计（默认开启）
SITE_DOMAIN localhost,127.0.0.1,... 影响 SPM 页面域名匹配与访问入口

API 示例（真实路径）

GET /spm/superpositionmodel/list?page=1&limit=10
GET /spm/visitlog/list?page=1&limit=20
GET /spm/superpositionmodel/info/1
GET /admin?spm=home.dashboard
{"code":0,"msg":"success","page":{"list":[...]}}

页面入口（templates）

  • /modules/spm/superpositionmodel -> templates/modules/spm/superpositionmodel.html
  • /modules/spm/visitlog -> templates/modules/spm/visitlog.html

权限码清单

spm:superpositionmodel:list|info|save|update|delete
spm:visitlog:list|info|save|update|delete

最小可跑通用例

  1. 确认账号具备 spm:superpositionmodel:list 和 spm:visitlog:list。
  2. 访问 /modules/spm/superpositionmodel，确认模型列表可加载。
  3. 调用 GET /spm/visitlog/list?page=1&limit=20，查看访问日志。
  4. 以带 spm 参数访问页面（如 /admin?spm=home.dashboard）后复查日志新增记录。

故障排查模板

现象 可能原因 处理建议

统计数据异常 指标口径不一致 统一口径并回溯计算逻辑
