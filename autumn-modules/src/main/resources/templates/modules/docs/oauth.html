认证授权（oauth/client）

本页用于说明 OAuth2 客户端接入、Token 管理与密钥配置。

混合加解密专项说明请查看：混合加解密（RSA+AES）章节。

功能范围

  • 客户端管理：/oauth/clientdetails/*
  • Token存储：/oauth/tokenstore/*
  • 加密密钥管理：/oauth/encryptkey/*
  • RSA握手接口：/rsa/api/v1/public-key、/client/public-key、/aes-key、/init

推荐文档结构

  • 接入前准备：client_id、redirect_uri、授权地址
  • 认证流程：授权码获取、Token 交换、Token 校验
  • 安全策略：密钥轮转、过期策略、回收策略

常见问题

  • 回调地址不匹配
  • Token 无效或过期
  • 跨系统认证失败
建议在本页后续加入时序图（授权码模式）与典型配置示例（开发/测试/生产）。

关键配置项（来自 sys_config.param_key）

配置键 默认值 说明
RSA_CONFIG {json} RSA 加密配置（密钥策略、过期策略等）
AES_CONFIG {json} AES 配置（key/vector 生成与有效期）
LOGIN_AUTHENTICATION oauth2:{clientId} 登录模式与默认client绑定
TOKEN_GENERATE_STRATEGY current Token 复用/重建策略

API 示例（真实路径）

GET /oauth/clientdetails/list?page=1&limit=10
POST /oauth/clientdetails/save {"clientId":"demo-client","clientSecret":"***"}
POST /rsa/api/v1/init {"session":"u-123","data":{"publicKey":"-----BEGIN PUBLIC KEY-----..."}}
POST /rsa/api/v1/endpoints {"session":"u-123"}

返回结构示例

{"code":0,"msg":"success","data":{"ras":{"publicKey":"..."}, "aes":{"key":"...","vector":"..."}, "endpoints":"..."}}

页面入口（templates）

  • /modules/oauth/clientdetails -> templates/modules/oauth/clientdetails.html
  • /modules/oauth/tokenstore -> templates/modules/oauth/tokenstore.html
  • /modules/oauth/encryptkey -> templates/modules/oauth/encryptkey.html

权限码清单

oauth:clientdetails:list|info|save|update|delete
oauth:tokenstore:list|info|save|update|delete
oauth:encryptkey:list|info|save|update|delete

最小可跑通用例

  1. 以管理员登录并确认权限 oauth:clientdetails:list。
  2. 访问 /modules/oauth/clientdetails，确认客户端列表可见。
  3. 调用 GET /oauth/clientdetails/list?page=1&limit=10，确认返回 code=0。
  4. 调用 POST /rsa/api/v1/init 完成一次握手，确认返回中包含 ras 与 aes 字段。

故障排查模板

现象 可能原因 处理建议
初始化加密失败 客户端公钥未上传或 session 不一致 先调用 /public-key 与 /client/public-key，并保持同一 session
Token 校验失败 密钥不一致或 Token 过期 检查密钥版本与过期策略
