防火墙（wall）

本页用于描述访问控制能力，包括 IP、URL、Host 与统计防护。

能力清单

  • 基础防火墙开关：GET/POST /wall/api/firewallOpen
  • 黑名单阈值管理：GET/POST /wall/api/firewallCount
  • 盾牌能力状态：/wall/api/shieldStatus、/shieldEnable、/shieldDisable
  • IP黑白名单：/wall/api/ipblack/*、/wall/api/ipwhite/*
  • 当前访问IP获取：/wall/api/currentIp

策略建议

  • 默认先白名单后黑名单，避免误放行
  • 上线前准备应急解封流程
  • 将高风险 URL 统一纳入黑名单策略

排查指南

  • 先检查 Shield 总开关状态
  • 再检查 IP 和 URL 命中规则
  • 最后结合访问统计确认真实来源
建议后续补充“策略优先级图”和“误拦截回滚步骤”，便于值班人员快速处理。

关键配置项（来自 sys_config.param_key）

配置键 默认值 说明
SITE_DOMAIN localhost,127.0.0.1,... 站点域名校验范围（域名不匹配时可被拦截）
BIND_DOMAIN 空 绑定域名白名单，绑定后防火墙放行
CLUSTER_ROOT_DOMAIN 站点域名推导 集群根域名，影响跨子域访问一致性

API 示例（真实路径）

GET /wall/api/ipblack/list?page=1&limit=10
POST /wall/api/ipblack/add?ip=1.2.3.4
POST /wall/api/ipwhite/remove?ip=1.2.3.4
POST /wall/api/shield/toggle

返回结构示例

{"code":0,"msg":"","count":2,"data":[{"ip":"1.2.3.4","count":10,"available":1}]}

页面入口（templates）

  • /modules/wall/ipblack -> templates/modules/wall/ipblack.html
  • /modules/wall/ipwhite -> templates/modules/wall/ipwhite.html
  • /modules/wall/urlblack -> templates/modules/wall/urlblack.html
  • /modules/wall/shield -> templates/modules/wall/shield.html
  • /modules/wall/host -> templates/modules/wall/host.html
  • /modules/wall/ipvisit -> templates/modules/wall/ipvisit.html

权限码清单

wall:ipblack:list|info|save|update|delete
wall:ipwhite:list|info|save|update|delete
wall:urlblack:list|info|save|update|delete
wall:host:list|info|save|update|delete, wall:ipvisit:list|info|save|update|delete
wall:shield:list|info|save|update|delete, wall:jump:list|info|save|update|delete

最小可跑通用例

  1. 以系统管理员登录（非管理员调用 /wall/api/* 会报错）。
  2. 访问 /modules/wall/ipblack，确认列表页可加载。
  3. 调用 POST /wall/api/ipblack/add?ip=1.2.3.4，确认返回 {"success":true}。
  4. 调用 GET /wall/api/ipblack/list?page=1&limit=10，确认能查到新增记录。

故障排查模板

现象 可能原因 处理建议

接口返回“系统错误” 当前登录用户不是系统管理员 确认用户具备管理员角色后再调用 /wall/api/*
