数据库运维（db）

用于管理数据库备份、备份策略、备份上传与恢复流程。

功能入口

  • 基础CRUD：/db/databasebackup/*、/db/databasebackupstrategy/*、/db/databasebackupupload/*
  • 统一备份中心：/db/backup/*（list/execute/download/restore/upload/chunk）
  • 任务控制：/db/backup/task/* 与 /db/backup/restore/*

推荐流程

  • 先制定策略，再执行备份任务
  • 备份文件统一归档并校验可恢复性
  • 恢复操作要求双人复核
建议补充“恢复演练模板”和“故障场景恢复RTO/RPO目标”。

关键配置/参数

参数 示例 说明
id 123 备份记录主键，用于下载/恢复/删除
taskKey restore-20260302-001 恢复任务标识，用于进度查询

API 示例（真实路径）

GET /db/backup/list?page=1&limit=10
POST /db/backup/execute {"strategyId":1}
GET /db/backup/download/123
POST /db/backup/restore/123
GET /db/backup/restore/progress/restore-20260302-001
{"code":0,"msg":"success","data":{"running":false}}

页面入口（templates）

  • /modules/db/databasebackup -> templates/modules/db/databasebackup.html
  • /modules/db/databasebackupstrategy -> templates/modules/db/databasebackupstrategy.html
  • /modules/db/databasebackupupload -> templates/modules/db/databasebackupupload.html

权限码清单

db:databasebackup:list|info|save|update|delete
db:databasebackupstrategy:list|info|save|update|delete
db:databasebackupupload:list|info|save|update|delete

最小可跑通用例

  1. 确认账号具备 db:databasebackup:list 与策略相关权限。
  2. 访问 /modules/db/databasebackup，查看备份列表。
  3. 调用 POST /db/backup/execute 执行一次备份任务。
  4. 调用 GET /db/backup/list?page=1&limit=10，确认新增备份记录。

故障排查模板

现象 可能原因 处理建议
备份失败 磁盘空间不足或权限不足 检查磁盘容量与目录权限
恢复失败 备份文件损坏或版本不兼容 校验备份完整性并核对数据库版本
