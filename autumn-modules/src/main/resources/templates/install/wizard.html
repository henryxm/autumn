欢迎使用

按步骤完成说明确认、数据库配置与检测。保存后将自动重启并连接您选择的数据库，完成表结构与基础数据准备。

说明与同意

  • 安装结束后，数据库连接会保存到程序目录下的 config/datasource.yml。如需改路径，可配置 autumn.install.config-path。
  • 数据库账号需具备建表、删表及日常读写权限。检测时会临时创建并删除一张测试表，属正常现象。
  • 为支持表情与多语言文本，向导会在常见数据库上附带合适的连接参数。若使用高级模式自写连接串，请确认字符集与库设置一致。
  • 重新安装：删除上述配置文件，或设置环境变量 AUTUMN_INSTALL_FORCE=true（或配置 autumn.install.force=true）。
  • 若只清空库中数据而保留配置文件，可能导致无法启动，请先恢复库或删除配置后重新安装。
  • 部分数据库需将官方驱动放入运行环境的 classpath，具体取决于部署方式。

数据库连接

按“数据库类型 → 连接方式 → 参数填写”的顺序配置，更贴近实际部署习惯。不同数据库会自动限制可用连接方式并给出建议。

先选择数据库类型后，再选择连接方式。

模式说明（点击表格行可快速切换）

模式 典型适用场景 与 Autumn 配合
MYSQL 推荐默认 希望与生产 MySQL 行为最接近；本地/演示环境跑通注解建表与常见 MySQL 语法。 JDBC URL 含 MODE=MySQL 时走框架内嵌 H2-MySQL 兼容 DDL 与方言路由，与业务以 MySQL 为基准的开发习惯一致。
NATIVE 谨慎 刻意验证 H2 原生语义；不依赖反引号、information_schema 等 MySQL 习惯用法。 按 DatabaseType.H2 走双引号 ANSI DDL；实体与手写 SQL 需与 ANSI/H2 语义对齐，勿假设 MySQL 专有写法。
POSTGRESQL 实验 尝试 PostgreSQL 风格函数或语法差异；小规模语法对照实验。 仍解析为 H2 类型，但 SQL 语义贴近 PG；与 Autumn 以 MySQL 为中心的默认路径不一致，仅适合有明确目标的场景。
ORACLE 实验 对照 Oracle 方言差异做本地实验（不等同于真实 Oracle）。 与 Autumn 默认 MySQL 族路径差异大，通常需配合高级 JDBC 或改写 SQL，不建议作为首次安装默认。
MSSQLSERVER 实验 对照 SQL Server 兼容语法做本地实验。 与 Autumn 默认栈差异大，多用于专项验证；生产请使用真实 SQL Server 与对应配置。
MARIADB 可选 希望略贴近 MariaDB 兼容语义时的本地尝试。 与 MYSQL 接近但仍属兼容层实验；若目标是生产 MariaDB，建议直接选数据库类型 MariaDB 而非 H2 模拟。
原生 H2 或非 MySQL 兼容模式下，不会启用 H2 + MODE=MySQL 适配链路，请确认业务 SQL 与方言匹配。
您选择了内存库：服务重启后数据会丢失，请勿用于正式环境。

Redis（可选）

不勾选则安装后以单机运行（配置中不写 spring.redis）；需要分布式缓存、会话或锁时再改配置并重启。

例如 PostgreSQL 需要 SSL 时可填 sslmode=require（以数据库文档为准）。
