欢迎使用，请先完成安装

下面几步会引导您同意说明、填写数据库信息、检查连接是否正常，最后保存设置。保存后程序会自动重启一次，然后为您建好数据表并准备初始数据。整个过程在网页上完成即可，无需编辑配置文件（除非您想改保存位置）。

说明与同意

  • 安装结束后，系统会把数据库连接信息保存到程序目录下的文件 config/datasource.yml（一般不用改）。若需改保存路径，可在配置里设置参数 autumn.install.config-path。
  • 您填写的数据库账号需要能新建表、删除表以及日常读写数据。检测步骤里会临时建一张测试表再删掉，这是正常现象。
  • 为支持昵称、备注里的表情符号（Emoji）等字符，向导在能自动处理的数据库上会带上合适的连接参数（例如 MySQL 会带上 connectionCollation=utf8mb4_unicode_ci，与常用线上配置一致）。若使用「高级模式」自己写连接地址，请确认库与连接串均为 UTF‑8 / 宽字符友好设置。
  • 需要重新安装时：删掉上面的 config/datasource.yml，或在启动前设置环境变量 AUTUMN_INSTALL_FORCE=true（参数名 autumn.install.force 也可在配置文件中设为 true）。
  • 若只删了数据库里的数据、但配置文件还在，程序可能起不来。请先恢复数据库，或删掉配置文件后重新走本安装页。
  • 部分数据库（如 Oracle、达梦、人大金仓）需要您把官方提供的驱动程序放到运行环境中，具体方式取决于您如何部署（例如与程序包放在同一目录的 lib 文件夹等）。

填写数据库信息

请选择数据库类型，并填写连接信息。若不清楚「主机、端口」可询问为您搭建数据库的服务商或同事。

会追加到连接地址末尾。例如 PostgreSQL 需要 SSL 时可填 sslmode=require（具体以数据库文档为准）。

检查连接与权限

将尝试连接数据库，并检查是否具备建表、删表权限。若失败，请根据提示联系管理员调整账号权限或核对主机、端口、库名、密码。

确认数据库现状

下方会列出当前连接下能读到的全部数据表及数量。表很多时加载可能稍慢，属正常现象。请确认是否要在当前库中继续安装（若已有同名表，可能与自动建表策略冲突，建议先备份或换库）。

  • 点击「确认并完成安装」后，会写入 config/datasource.yml，随后程序自动重启。
  • 重启成功后将按系统设置自动维护表结构（与配置项 autumn.table 等相关），并写入网站运行所需的基础数据。

安装已完成

程序正在重启，请等待几秒再访问。若长时间打不开页面，请查看运行日志或联系运维（例如是否使用了守护进程自动拉起程序）。
